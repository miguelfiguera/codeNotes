## Design Doc.

[Design Docs Explained](https://www.youtube.com/watch?v=bgHL41e7vgI)

Is just a doc with some special items to talk about and develope a conversation between engineers and product managers in a project.

1. Abstract: Summary of the feature aspects, advantages, important stuff related to the impact of this feature.

2. UX changes: This defines the brand new or modified ux changes for the users and what they should expect, also what they will be able to do.

3. UI changes: this defines the changes in the UI to make this feature work as it should be expected.

4. Comments: Space to talk about the problems that could develope from this feature.

5. API:

6. All other topics that you find important
